# docker-deluge

A Docker image that runs Deluge as a single container, designed to be deployed as a Kubernetes StatefulSet with a persistent config volume.

The image bundles:

- **Deluge 2.2.0** (installed via pip on Alpine)
- **libtorrent-rasterbar 2.0.10** (from the `emmercm/libtorrent` base image)
- **[ltConfig 2.0.0](https://github.com/ratanakvlun/deluge-ltconfig)** plugin
- A legacy MaxMind GeoIP country database

Configuration is driven by environment variables: two small Python scripts run at container start and merge `DELUGE_CONF_CORE_*` / `DELUGE_CONF_WEB_*` env vars into `core.conf` and `web.conf` without overwriting existing values.

## Image tags

CI publishes three tags per push to `main`:

| Tag | Example | Purpose |
|---|---|---|
| `latest` | `latest` | Mutable rolling tag |
| `<deluge-version>` | `2.2.0` | Pinned to the Deluge version in the Dockerfile |
| `<deluge-version>-<shortsha>` | `2.2.0-a57047a` | Immutable snapshot |

## Ports

| Port | Purpose |
|---|---|
| `8112/tcp` | Deluge web UI |
| `58846/tcp` | Daemon RPC |
| `61534 + pod_index` (tcp+udp) | BitTorrent listen — assigned at runtime from `POD_NAME` (see below). Cannot be expressed via Docker `EXPOSE`. |

## Volumes

| Path | Owner | Purpose |
|---|---|---|
| `/home/deluge/.config/deluge` | `deluge` (uid 1000) | Deluge config dir (`core.conf`, `web.conf`, `state/`, `torrents/`, plugins). Mount your PVC here. |

## Environment variables

### Generic config injection

Any environment variable prefixed with `DELUGE_CONF_CORE_` or `DELUGE_CONF_WEB_` becomes a key in `core.conf` / `web.conf` respectively. The remainder of the name is lowercased.

Type conversion:

- `true` / `yes` → boolean `True`
- `false` / `no` → boolean `False`
- `[a,b,c]` → list `['a', 'b', 'c']` (core only)
- Anything else → string

Examples:

```
DELUGE_CONF_CORE_DOWNLOAD_LOCATION=/downloads
DELUGE_CONF_CORE_MAX_UPLOAD_SPEED=500
DELUGE_CONF_CORE_RANDOM_PORT=false
DELUGE_CONF_WEB_PORT=8112
```

### Per-pod listen port

If `POD_NAME` is set and matches `*-<int>$` (e.g. `deluge-0`, `deluge-1`), the BitTorrent listen port is set to `61534 + pod_index`. Wire this via the Kubernetes downward API:

```yaml
env:
  - name: POD_NAME
    valueFrom:
      fieldRef:
        fieldPath: metadata.name
```

> Note: Deluge's `random_port` defaults to `true`, which overrides the assigned listen port. To make the per-pod port actually bind, also set `DELUGE_CONF_CORE_RANDOM_PORT=false`.

### Web UI password

| Variable | Effect |
|---|---|
| `DELUGE_WEB_PASSWORD` | Sets the web UI password (salted SHA-1, the format Deluge expects). Skips the first-login prompt. |
| `DELUGE_WEB_SALT` | Optional. Use a fixed salt instead of the random one generated by Deluge. |

### Other

| Variable | Default | Purpose |
|---|---|---|
| `DELUGE_LOGLEVEL` | `info` | Passed to both `deluged` and `deluge-web`. |

## Kubernetes notes

- **Run as uid/gid 1000.** Set `securityContext.runAsUser: 1000`, `runAsGroup: 1000`, and `fsGroup: 1000` so the mounted PVC is writable.
- **`POD_NAME`** must be wired via the downward API for per-pod port assignment.
- **Graceful shutdown** is supported — the entrypoint forwards `SIGTERM`/`SIGINT` to both `deluged` and `deluge-web`, so torrents are checkpointed before the kubelet's grace period expires.
- **Downloads directory** is not declared as a volume. If you mount one (e.g. `/downloads`), set `DELUGE_CONF_CORE_DOWNLOAD_LOCATION=/downloads` and make sure it's writable by uid 1000.
- **Probes:** TCP readiness on `8112` or `58846` is reasonable.

## Local build & run

```sh
docker build -t deluge:dev .

docker run --rm -it \
  -e POD_NAME=deluge-0 \
  -e DELUGE_WEB_PASSWORD=changeme \
  -e DELUGE_CONF_CORE_RANDOM_PORT=false \
  -v deluge-config:/home/deluge/.config/deluge \
  -p 8112:8112 -p 58846:58846 -p 61534:61534 \
  deluge:dev
```

Then open <http://localhost:8112>.

## Files

| File | Purpose |
|---|---|
| `Dockerfile` | Image build (single stage on `emmercm/libtorrent:2.0.10-alpine`). |
| `entrypoint.sh` | Runs the config injectors, copies bundled plugins, starts `deluged` and `deluge-web`, forwards signals. |
| `inject_core_config.py` | Merges `DELUGE_CONF_CORE_*` env vars into `core.conf`. Also handles `POD_NAME` listen-port assignment. |
| `inject_web_config.py` | Merges `DELUGE_CONF_WEB_*` env vars into `web.conf`. Handles `DELUGE_WEB_PASSWORD` / `DELUGE_WEB_SALT`. |
| `.github/workflows/docker.yml` | Builds and pushes to Docker Hub on every push to `main`. |
